# Лабораторная работа №2: Локализация WPF-приложения

**Тема:** Реализация многоязычной поддержки в WPF-приложении с использованием различных подходов.

**Цель:** Закрепить практические навыки локализации WPF-приложения, изучив и применив три различных подхода к работе с переводами: использование файлов ресурсов (RESX), словарей ресурсов XAML и внешней библиотеки классов.

**Исходные данные:** WPF-приложение, разработанное в рамках **Лабораторной работы №1**. Предполагается, что в нем есть как минимум одно окно с интерфейсом, содержащим текстовые элементы (надписи, заголовки, содержимое кнопок, сообщения).

---

## Общие требования

1. При разработке необходимо соблюдать паттерн MVVM.
2. Разработка должна вестись в системе контроля версий.
3. Программное средство должно использовать уникальные стили (разрешается использование сторонних библиотек).
4. По результатам работы должен быть написан отчёт.
5. Название файла отчёта должно соответствовать формату `ЛР_2_Иванов_Иван.docx`.

---

## Задание

1. Необходимо модифицировать приложение из лабораторной работы №1, добавив поддержку **двух языков**: русского (язык по умолчанию) и английского. Переключение языка должно происходить **динамически** во время работы приложения (например, через выбор в ComboBox или меню) без его перезапуска.

2. Реализовать перевод всех статических текстовых элементов интерфейса и как минимум одного строкового сообщения, выводимого в процессе работы (например, в MessageBox или Label), **тремя разными способами**:

   - **Использование файлов ресурсов (RESX)**
   - **Использование словарей ресурсов XAML (ResourceDictionary)**
   - **Использование внешней библиотеки**

3. Каждый вариант перевода должен быть выполнен в отдельной ветке.

---

## Структура отчёта

1. **Введение** (цели и задачи)
2. **Теория** — краткое описание вариантов перевода.
3. **Варианты перевода**

   Для каждого перевода:

   a. Описание механизма работы перевода с примерами, относящимися к вашему приложению.

   b. Скриншоты, показывающие механизм работы.

4. **Ссылка на проект** в системе контроля версий.
5. **Заключение.**
